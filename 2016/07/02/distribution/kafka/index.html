Log Compaction

A log is a data structure that maintains a timeline of things.

Here is a high-level picture that shows the logical structure of a Kafka log with the offset for each message.

The head of the log is identical to a traditional Kafka log.

Log compaction adds an option for handling the tail of the log.

In 0.7 the tail of the log is always deleted.

In 0.8 there is a per-topic option to choose either "delete" or "compact".

configuration description default
log.cleaner.enable Turn on log cleaning. This is disabled by default for now. false
log.cleaner.dedupe.buffer.size The amount of memory that can be used for log cleaning across all cleaner threads. 500 MB
log.cleaner.io.buffer.size The buffer used for reads and writes to the log. 512K
log.cleaner.delete.retention.ms The time for which delete markers are retained in the tail of the log. 24 Hours
log.cleaner.min.cleanable.ratio The minimum ratio of cleanable log head to total log for a log to be eligible for cleaning. 0.5
log.cleaner.threads The number of background threads to use for log cleaning. 1
log.cleaner.max.io.bytes.per.second The maximum I/O that the log cleaning across all threads. Infinity
log.cleaner.backoff.ms The time to wait in between checking if there are any cleanable logs. 30 seconds
log.cleanup.policy The default policy for handling log tails. Can be either delete or compact. delete
